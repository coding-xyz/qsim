    qsim.analysis.error_budget¶

    qsim.analysis.error_budget ¶

    Legacy error-budget report builders for simulation observables.

    build_report(observables) ¶

    Build a lightweight error-budget report from observables.

    The current model is heuristic and deterministic. It is intended as a stable baseline for regression checks before introducing richer physical decomposition models.

    参数：

    名称 类型 描述 默认
    observables Observables

    Aggregated observables from a simulation trace.

    		 必需

    返回：

    类型 描述
    Report

    A Report with:
    Report
    • summary: high-level status and fidelity proxy.
    Report
    • error_budget: dephasing/leakage-like terms in [0, 1].

    qsim.analysis.error_budget_pauli ¶

    build_component_budget(*, baseline_scaling, component_model, ablation_scaling) ¶

    Build Pauli+ component error budget from baseline vs component-off ablations.

    Contribution for each component is defined as: delta_inverse_lambda = (1/lambda_baseline) - (1/lambda_component_off).

    write_component_ablation_csv(*, component_model, budget, out_path) ¶

    Write component-level ablation table for quick inspection/export.

    qsim.analysis.passes ¶

    Composable analysis-pass helpers used by the workflow runner.

    AnalysisPass ¶

    Bases: Protocol

    Callable protocol for analysis pass implementation.

    An analysis pass consumes Trace + ModelSpec and returns a plain dictionary payload that can be serialized by the workflow layer.

    qsim.analysis.pauli_plus¶

    qsim.analysis.pauli_plus ¶

    Pauli-plus component modeling and scaling sweep helpers.

    build_component_error_model(*, logical_x, logical_z, mean_excited, final_p1) ¶

    Build component-level error seeds from current run metrics.

    This is a bridge model for offline analysis. It converts available workflow observables/logical-error summaries into component bins expected by Pauli+ scaling and ablation routines.

    build_scaling_report(epsilon_by_d) ¶

    Build scaling report with Lambda3/5 from epsilon-by-distance results.

    run_pauli_plus_sim(model_spec, *, qec_engine, code_distance, shots, seed, options=None) ¶

    Run one code-distance Pauli+ simulation through selected engine.

    For qec_engine='auto', this function tries Stim first and falls back to Cirq if Stim returns fallback quality.

    run_scaling_sweep(*, qec_engine, component_errors, code_distances, shots, seed, options=None) ¶

    Run Pauli+ simulations across code distances and return scaling report.

    Includes run-level backend quality statistics: native_runs, fallback_runs, native_runs_ratio.

    build_sensitivity_report(observables, logical_error, *, seed, sweep=None) ¶

    Build deterministic sensitivity summary for P1-M4.

    This version uses a stable proxy model to estimate local sensitivities from current run metrics. It favors reproducibility over physical completeness and is suited for ranking and regression tracking.

    参数：

    名称 类型 描述 默认

    		 必需

    Logical error summary from decoder outputs.

    		 必需

    Run seed recorded for reproducibility.

    		 必需

    Optional declared sweep configuration from backend settings.

    		 None

    返回：

    类型 描述

    dict

    qsim.analysis.trace_semantics¶

    qsim.analysis.trace_semantics ¶

    annotate_trace_metadata(trace, *, num_qubits=None, dimension=None, engine_name=None) ¶

    Attach canonical state semantics metadata to a trace in-place.

    extract_p1_series(trace) ¶

    Extract a semantically safe single-qubit p1(t) series from a trace.

    Supported encodings: - per_qubit_excited_probability: use row[0] for single-qubit case. - basis_population_single_qubit: use row[1].

    返回：

    类型 描述
    list[float]

    A list of p1 values aligned with trace.times.

    引发：

    类型 描述
    ValueError

    If the trace encoding cannot be safely interpreted as a single-qubit p1(t) series.

    infer_state_encoding(trace, *, num_qubits=None, dimension=None, engine_name=None) ¶

    Infer a safe, explicit state encoding label for a trace.

    The labels intentionally prefer ambiguous_* over aggressive guesses when multiple interpretations are plausible.

    Return whether two traces support pointwise numeric comparison.
